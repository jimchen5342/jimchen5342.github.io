  2. 實例化(new)：它會建立新的空白物件。 然後，會執行緊接在後面的函數呼叫(建構函式)， 並將新的空物件設定為該函數內的 "this" 值。
    每次 new 一個物件的時候，物件裡所有的屬性都會被 new 一份出來， 但 prototype(原型)就不會，因為是「繼承」的概念，所以就是大家共用一份。
  3. 存取：在 JavaScript 中，物件只是名稱/值配對的集合(屬性及方法) -- 可以把 JavaScript 物件想像成含有字串索引鍵的字典。
